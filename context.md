## 🗄 Database Schema

### Product Model
```python
class Product(models.Model):
    CATEGORY_CHOICES = [
        ('INDOORS', 'Indoors'),
        ('OUTDOORS', 'Outdoors'),
        ('PARTS', 'Parts'),
        ('RFSPORTS', 'RF Sports'),
    ]
    name = models.CharField(max_length=200)
    category = models.CharField(max_length=50, choices=CATEGORY_CHOICES, default='INDOORS')
    price = models.DecimalField(max_digits=14, decimal_places=2)
    discount_price = models.DecimalField(max_digits=14, decimal_places=2, null=True, blank=True)
    description = models.TextField()
    image_file = models.ImageField(upload_to='products/', null=True, blank=True)
    image_url = models.URLField(max_length=1000, null=True, blank=True)  # Supabase URL stored here
    sku = models.CharField(max_length=50, null=True, blank=True)
    stock = models.IntegerField(default=10)
    source = models.CharField(max_length=20, choices=[('admin', 'Admin'), ('catalogue', 'Catalogue')], default='admin')
    needs_image = models.BooleanField(default=False)
    created_at = models.DateTimeField(auto_now_add=True)
    
    @property
    def display_image(self):
        """Returns image_file.url > image_url > fallback Unsplash image"""
```

### Inquiry Model
```python
class Inquiry(models.Model):
    STATUS_CHOICES = [
        ('NEW', 'New'),
        ('CONTACTED', 'Contacted'),
        ('CLOSED', 'Closed'),
    ]
    product = models.ForeignKey(Product, on_delete=models.CASCADE, related_name='inquiries')
    name = models.CharField(max_length=200)
    contact_number = models.CharField(max_length=20)
    quantity = models.PositiveIntegerField(default=1)
    status = models.CharField(max_length=20, choices=STATUS_CHOICES, default='NEW')
    created_at = models.DateTimeField(auto_now_add=True)
```

```
Product ──→ Inquiry (1:N, cascade delete)
```

## 🧠 Core Views Logic (`backend/products/views.py`)

### Authentication & Session Management

**Admin Login**: Checks `.env` ADMIN_EMAIL/ADMIN_PASSWORD first, sets `request.session['is_admin'] = True`, redirects to admin dashboard. Admin is session-based (NOT Django auth).

**Regular User Auth**: Looks up user by email in DB, then authenticates with username via `authenticate()`. Supports `next` parameter (both GET and POST) for redirect-after-login.

**Logout**: Flushes session data via `session.flush()` and standard Django `logout(request)`.

*(Note: AdminRestrictMiddleware has been removed from settings and code).*

### Cart System (Session-based)
```python
# Cart stored in session as: { 'product_pk_string': quantity_int }
# Helper function: get_cart_data(request) -> (cart_items_list, subtotal)
```
- **Add**: POST only, increments quantity, clamped to stock level.
- **Remove**: POST only, deletes key from session dict.
- **Update**: POST only, validates quantity, clamps to stock.
- **Submit Inquiry Button**: Checks if user is authenticated/admin. If authenticated, redirects to `home`. If not, shows an auth modal prompting redirection to login.

### Inquiry System (Quote Requests)
Instead of a checkout/order payment pipeline, users make inquiries about products:
1. In `product_detail.html`, users select a quantity and click **Submit Inquiry**.
2. This opens `#inquiryModal` to collect their Name and Contact Number.
3. Submitting the form triggers an AJAX POST to `/inquiry/submit/` which validates input and records an `Inquiry` row linked to the product.
4. The user is notified of a successful submission using a Toast animation.

### Admin Dashboard
A 3-tab dashboard for staff:
- **Products**: Table of all products with image previews, category tags, stock, and deletion triggers.
- **Inquiries**: Review quote requests, filterable by status (`NEW`, `CONTACTED`, `CLOSED`). Allows updating inquiry status (Advance/Cancel) or deleting them.
- **Sales Reports**: Displays simulated billing dashboards (Total revenue, Total orders, Monthly bar heights, Best seller) along with actual count of `active_quotes` (NEW inquiries).

### AI Chat (Mohanlal Mascot)
- Groq API integration with `llama-3.3-70b-versatile` (fallback to `llama-3.1-8b-instant`).
- Conversational context: last 6 messages preserved in history.
- System prompt instructs Mohanlal to act as a warm, knowledgeable mascot for playground equipment.
- Directly prints the support helpline number (+91 9924343003) for complex/bulk requirements.

### Image Upload to Supabase
- Uploads to: `{SUPABASE_URL}/storage/v1/object/{bucket}/{filename}` via direct HTTP calls.
- Returns public URL: `{SUPABASE_URL}/storage/v1/object/public/{bucket}/{filename}`
- Falls back to local disk storage if Supabase credentials are not found.
- Image display priority property:
  1. `image_file` (local media/ upload)
  2. `image_url` (external URL, including Supabase public URL)
  3. Fallback Unsplash image

## 🎯 Important Implementation Details

1. **Admin vs Regular User**:
   - Admin: Session-based (`request.session['is_admin']`), logins check credentials from environment `.env`. Accesses admin panel.
   - Regular User: Django database Auth (`request.user.is_authenticated`).
2. **Double Home Routing**:
   - `urls.py` contains duplicate root patterns. The `views.home_view` acts as the active handler and passes the featured product context.
3. **Cart & Inquiry Workflow**:
   - Cart stores products and quantities in standard sessions.
   - Inquiry forms trigger AJAX submissions to store contact info, showing a Toast popup on success.
4. **Supabase Integration**:
   - Product uploads are directed using pure `requests` via Supabase JSON APIs to save overhead, saving files in the default public bucket.

3. **Cart Inquiry Redirection**: Clicking "Submit Inquiry" in `cart.html` when logged in redirects the user back to the home page instead of sending all items as a batch inquiry.
