### 1E. Autocomplete Search Efficiency

**Current state** (`backend/products/search.py`): Uses `Q(name__icontains=q) | Q(description__icontains=q) | Q(sku__icontains=q)` â€” full-text scan on every keystroke.

**Files to modify**: `backend/products/search.py`, `backend/products/views.py`, `backend/products/urls.py`

**Plan**:

## Phase 7: Deployment / Infra

### 7A. Vercel Static Asset Caching

**Current state** (`vercel.json`): Static routes have NO `Cache-Control` headers:
```json
{
  "src": "/static/(.*)",
  "dest": "frontend/static/$1"
}
```

**Fix**: Add `Cache-Control` headers:
```json
{
  "src": "/static/(.*)",
  "headers": {
    "Cache-Control": "public, max-age=31536000, immutable"
  },
  "dest": "frontend/static/$1"
}
```

Since WhiteNoise's `CompressedManifestStaticFilesStorage` adds content hashes to filenames, long cache times are safe.

### 7B. WhiteNoise in Static Route

The Vercel static route serves files directly from `frontend/static/` without going through WhiteNoise. For hashed files, this is fine. For non-hashed files (theme.css, etc.), we should route through Django/WhiteNoise. However, since theme.css already has `?v=20260718` cache-busting, this is acceptable.

**No change needed** â€” the current setup is functional.
